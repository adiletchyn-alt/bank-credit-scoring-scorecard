# 🏦 Автоматизированная система кредитного скоринга (Credit Scoring App)

Репозиторий содержит сквозной аналитический проект по разработке классической банковской скорокарты (Scorecard) для оценки вероятности дефолта заемщиков потребительских кредитов.

**Статус проекта:** Завершен. На базе финальной математической модели развернуто интерактивное веб-приложение для кредитных инспекторов.

---

## 📊 Бизнес-задача
Автоматизация процесса андеррайтинга (принятия решений о выдаче кредита) для снижения уровня дефолтов (рисков) банка и сокращения времени обработки кредитных заявок при сохранении стабильного объема выдачи.

---

## 🛠️ Архитектура проекта и Стек технологий
* **Язык разработки:** Python 3.14
* **Библиотеки:** `pandas`, `numpy` (обработка и очистка данных), `scikit-learn` (машинное обучение), `openpyxl` (работа с Excel-артефактами), `streamlit` (веб-интерфейс).

Проект реализован в виде модульной структуры:
1. **Предобработка и EDA:** Очистка аномалий (удаление ошибочного возраста < 18 лет, изоляция технических кодов просрочек `96` и `98`).
2. **Биннинг и Расчет WoE/IV:** Оптимальное разбиение непрерывных признаков на интервалы. Расчет *Weight of Evidence* (веса риска групп) и *Information Value* (информативности факторов).
3. **Отбор признаков:** Исключение нерелевантных признаков с IV < 0.02 (например, `NumberRealEstateLoansOrLines`).
4. **Моделирование:** Обучение логистической регрессии на трансформированных WoE-данных.
5. **Масштабирование (Scaling):** Перевод логарифмических коэффициентов в стандартные банковские баллы (Target Score: 600, PDO: 20).
6. **Интерфейс:** Создание симулятора кредитного конвейера на Streamlit.

## 📈 Ключевые результаты и метрики

### 1. Качество модели
* **Метрика качества Gini (Джини):** 
  * На обучающей выборке (Train): **0.5721**
  * На тестовой выборке (Test): **0.5747**
* **Вывод:** Минимальный разрыв метрик (менее 0.3%) доказывает абсолютную стабильность модели и отсутствие переобучения (overfitting). Показатель Джини > 0.55 является высоким стандартом для розничного кредитования.

### 2. Финальная скорокарта (Пример из credit_scorecard.xlsx)
Модель выявила логичные и устойчивые бизнес-зависимости. Надежность клиентов (и их скоринговый балл) растет пропорционально увеличению возраста и уровня доходов:

| Признак | Группа (Бакет) | WoE | Балл (Score) |
| :--- | :--- | :---: | :---: |
| **Возраст** | (21.0, 39.0] | -0.4966 | **48** |
| **Возраст** | (65.0, 109.0] | 1.0637 | **67** |
| **Доход** | (-0.001, 3000.0] | -0.3321 | **50** |
| **Доход** | (9083.0, 3008750.0] | 0.4177 | **59** |

## 🚀 Инструкция по запуску локально

1. Клонируйте репозиторий и перейдите в папку проекта.
2. Установите зависимости:
